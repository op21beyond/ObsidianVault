
# 목표 
* Architecture 문서에서 모든 NPU 명령어에 대한 허용된 모든 필드의 조합을 테이블 1개로 만들어야 함
* NPU를 시뮬레이션 검증하기 위한 Test Case 생성을 위한 것이지만, Verilog Code 생성은 불필요함.
# 입력
* Architecture 문서는 텍스트 기반 PDF 포맷. 300페이지 정도 분량. Taged PDF인지는 모름. 문서 내 포함된 표 또는 수식, 이미지 등은 원본에서 이미지 형태로 복사되어 붙여져 있을 수 있음
* NPU 명령어의 필드 구성은 원본은 엑셀로 작성되어 있고 이용 가능함 (나중에 검증 옵션으로만 생각하고 가급적 PDF로만 처리해야 함)
# 출력
* 엑셀 또는 CSV 포맷
# 상세 설명

* 최종 table의 행은 NPU 명령어, 열은 각 필드의 값이며 하나의 Test Case가 됨
* 모든 명령어들은 unique한 필드 이름 집합의 일부분을 조합하여 구성되는데, Table의 행은 전체 필드들을 포함하며 각 Test Case에서는 어떤 필드들은 사용되지 않는 경우 Don't Care값을 가짐
* NPU 명령어들은 문서 안에서 워드(word), 비트(bit) 표시와 함께 필드 이름들이 포함된 표(삽입된 이미지)안에 기술되어 있음
* 각 명령어들은 여러 필드들로 구성됨
* 특정 필드의 값에 따라 다른 필드들의 구성이 달라질 수 있음
* 각 필드들은 데이터 타입과 허용 값의 형태와 범위를 가짐 
* 각 필드들은 명령어 별로 제약 조건이 있을 수 있음 
* 필드들의 조합에 제약 조건이 있을 수 있음
* 문서에서 제약에 대한 기술은 여러 군데 흩어져 있을 수 있음
* 제약을 표시하는 특별한 문단 구조를 갖고 있지 않음
# 구현 가이드
* 모든 명령어의 리스트를 추출 
* 모든 데이터 타입의 종류와 허용된 값을 만들 수 있는 정보 추출
* 모든 명령어별 필드 구성을 표에서 추출
* 모든 필드들을 추출한 다음 unique한 필드 이름 집합 생성
* 각 필드들의 데이터 타입 정보 추출
* 각 필드들의 제약 사항 추출
* 제약 사항은 특정한 위치나 문단 구조를 갖고 있지 않고 제약 사항의 종류도 여러가지일 수 있음
* 필드들간의 의존정 제약사항도 추출
